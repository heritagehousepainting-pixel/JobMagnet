# SQLite → Render Postgres Migration Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Move JobMagnet from a single-file SQLite database to managed Render Postgres, delivering durability (managed backups/PITR) and scale (concurrent writers, workers > 1) without changing the `db.py` public API.

**Architecture:** Swap the connection backend in one seam (`db.get_conn` / `db_core`), port all raw SQL (5 files) from SQLite dialect to Postgres, point the test suites at an ephemeral Postgres, and provision managed Postgres in `render.yaml`. No ORM, no schema redesign, no data migration (dev data is disposable).

**Tech Stack:** Python 3.12, Flask, `psycopg` v3 (raw SQL, no ORM), Render Postgres, gunicorn.

## Global Constraints

- Python 3.12.7 (matches `render.yaml`).
- No ORM — keep raw SQL throughout.
- Light dependencies — only add `psycopg[binary]` (v3).
- Tests are framework-free: plain scripts, `exit 0` = pass. Run with `./.venv/bin/python <file>`.
- **All 453 checks must pass on Postgres** (406 in `test_smoke.py`, 47 in `test_compliance_core.py`) — this is the acceptance gate.
- `db.py` public function API is unchanged — downstream callers (`app.py`, `assistant.py`) keep working.
- Keep existing 0/1 integer "boolean" columns as integer (e.g. `resolved=0`) — do NOT convert to SQL `boolean`; it would break app-level comparisons.
- Keep timestamp columns as TEXT storing `now_iso()` (UTC ISO-8601) — lexical comparison stays valid; avoids timestamptz churn.
- Per-call connections (open in `get_conn()`, closed at each existing `conn.close()` site). No connection pool in this plan (pooling = documented later optimization).

**Raw SQL is confined to 5 files:** `db_core.py`, `db.py`, `app.py`, `assistant.py`, `consent.py`. (`roi.py`, `tc_messaging.py`, `reviewsync.py` use `db.py` helpers and contain no raw SQL — the spec over-listed them.)

**Dialect transformation rules (apply everywhere):**

| SQLite | Postgres (psycopg v3) |
|---|---|
| `?` placeholder | `%s` |
| `cur.lastrowid` | `INSERT … RETURNING id`, then `cur.fetchone()["id"]` |
| `INTEGER PRIMARY KEY AUTOINCREMENT` | `BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY` |
| `INSERT OR IGNORE` | `INSERT … ON CONFLICT DO NOTHING` (none currently present) |
| `datetime('now','-7 days')` etc. (SQL-side date math) | compute cutoff in Python via `now_iso()` math, pass as `%s` param |
| `PRAGMA journal_mode=WAL` | delete |
| `PRAGMA table_info(t)` | `SELECT column_name FROM information_schema.columns WHERE table_name=%s` |
| `sqlite3.IntegrityError` | `psycopg.errors.UniqueViolation` + `conn.rollback()` before reuse/close |
| `c.executescript("…")` | `conn.execute("…")` (psycopg runs multi-statement strings when no params bound) |

- [ ] **Step 2: Fix `db_core.py` import + IntegrityError + lastrowid**

```python
def create_user(email, password_hash, business_id):
    """Create a login. Returns the new user id, or None if the email is taken."""
    conn = get_conn()
    try:
        cur = conn.execute(
            "INSERT INTO users (email, password_hash, business_id, created_at) "
            "VALUES (%s,%s,%s,%s) RETURNING id",
            (email.strip().lower(), password_hash, business_id, now_iso()))
        uid = cur.fetchone()["id"]
        conn.commit()
    except psycopg.errors.UniqueViolation:
        conn.rollback()
        uid = None  # email already registered
    conn.close()
    return uid
```
Then convert the other `db_core.py` queries' `?` → `%s` and any `.lastrowid` → `RETURNING id` + `cur.fetchone()["id"]` (e.g. `log_turn`).

Replace lines 9-24 (the `tempfile` block) with a throwaway-database bootstrap. It creates a uniquely-named database off `TEST_DATABASE_URL`, points `DATABASE_URL` at it, and registers cleanup:
```python

import sys
import uuid
import atexit
import psycopg

# Run against a throwaway Postgres DB so the suite is isolated + idempotent and
# never touches real data. Create a uniquely-named DB, point the app at it, drop
# it on exit. Set env BEFORE importing config/app.
_ADMIN_URL = os.environ["TEST_DATABASE_URL"]
_TEST_DB = "jm_test_" + uuid.uuid4().hex[:12]
_admin = psycopg.connect(_ADMIN_URL, autocommit=True)
_admin.execute(f'CREATE DATABASE "{_TEST_DB}"')
_admin.close()
# Build the app-facing URL by swapping the database name on the admin URL.
import urllib.parse as _u
_p = _u.urlparse(_ADMIN_URL)
_APP_URL = _p._replace(path="/" + _TEST_DB).geturl()
os.environ["DATABASE_URL"] = _APP_URL

@atexit.register
def _drop_test_db():
    a = psycopg.connect(_ADMIN_URL, autocommit=True)
    a.execute("SELECT pg_terminate_backend(pid) FROM pg_stat_activity "
              "WHERE datname=%s AND pid<>pg_backend_pid()", (_TEST_DB,))
    a.execute(f'DROP DATABASE IF EXISTS "{_TEST_DB}"')
    a.close()

os.environ["JOBMAGNET_PROVIDER"] = "demo"
os.environ["JOBMAGNET_OWNER_PASSWORD"] = "jobmagnet123"
os.environ["JOBMAGNET_SECRETS_KEY"] = ""
```

- [ ] **Step 1: Port `_ensure_columns` off PRAGMA**
```python
def _ensure_columns(cursor, table, cols):
    """Add any missing columns to an existing table (simple forward migration).
    `cols` maps column name -> SQL type."""
    rows = cursor.execute(
        "SELECT column_name FROM information_schema.columns WHERE table_name=%s",
        (table,)).fetchall()
    have = {r["column_name"] for r in rows}
    for name, sqltype in cols.items():
        if name not in have:
            cursor.execute(f"ALTER TABLE {table} ADD COLUMN {name} {sqltype}")
```

Enumerate: `git grep -nE "datetime\(|strftime|julianday|'now'" db.py consent.py app.py assistant.py`
